# 快速启动指南

## 一、前置要求

1. **Node.js** v18 或更高版本
2. **npm** 或 **yarn**
3. **Claude API密钥**（从 Anthropic 获取）

## 二、快速启动步骤

### 步骤1：安装后端依赖

```bash
cd backend
npm install
```

### 步骤2：配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cd ..
cp .env.example .env
```

编辑 `.env` 文件，填入你的 Claude API 密钥：

```
ANTHROPIC_API_KEY=your_claude_api_key_here
PORT=3000
NODE_ENV=development
```

### 步骤3：启动后端服务

```bash
cd backend
node server.js
```

看到以下输出表示启动成功：
```
🚀 服务器已启动
📍 地址: http://localhost:3000
```

### 步骤4：打开前端

有两种方式：

**方式1：直接打开HTML文件**
- 在浏览器中打开 `frontend/index.html`
- 注意：由于CORS限制，某些功能可能无法正常工作

**方式2：使用本地服务器（推荐）**

```bash
# 使用Python
cd frontend
python -m http.server 8080

# 或使用Node.js
npx http-server frontend -p 8080
```

然后访问 `http://localhost:8080`

## 四、常见问题

### Q1: 后端启动失败
**A:** 检查：
- Node.js版本是否 >= 18
- 是否已安装依赖（`npm install`）
- 端口3000是否被占用

### Q2: API调用失败
**A:** 检查：
- `.env` 文件中的 `ANTHROPIC_API_KEY` 是否正确
- API密钥是否有效
- 网络连接是否正常

### Q3: 前端无法连接后端
**A:** 检查：
- 后端服务是否已启动
- `frontend/js/api.js` 中的 `API_BASE_URL` 是否正确
- 浏览器控制台是否有CORS错误

### Q4: 数据库错误
**A:** 检查：
- `database/` 目录是否有写入权限
- SQLite是否正常工作

## 五、开发模式

使用 `nodemon` 自动重启（需要先安装）：

```bash
cd backend
npm install -g nodemon
npm run dev
```
